      45              : //
      46              : // The ~uvm_root~ class serves as the implicit top-level and phase controller for
      47              : // all UVM components. Users do not directly instantiate ~uvm_root~. The UVM
      48              : // automatically creates a single instance of <uvm_root> that users can
      49              : // access via the global (uvm_pkg-scope) variable, ~uvm_top~.
      50              : //
      51              : // (see uvm_ref_root.gif)
      52              : //
      53              : // The ~uvm_top~ instance of ~uvm_root~ plays several key roles in the UVM.
      54              : //
      55              : // Implicit top-level - The ~uvm_top~ serves as an implicit top-level component.
      56              : // Any component whose parent is specified as ~null~ becomes a child of ~uvm_top~.
      57              : // Thus, all UVM components in simulation are descendants of ~uvm_top~.
      58              : //
      59              : // Phase control - ~uvm_top~ manages the phasing for all components.
      60              : //
      61              : // Search - Use ~uvm_top~ to search for components based on their
      62              : // hierarchical name. See <find> and <find_all>.
      63              : //
      64              : // Report configuration - Use ~uvm_top~ to globally configure
      65              : // report verbosity, log files, and actions. For example,
      66              : // ~uvm_top.set_report_verbosity_level_hier(UVM_FULL)~ would set
      67              : // full verbosity for all components in simulation.
      68              : //
      69              : // Global reporter - Because ~uvm_top~ is globally accessible (in uvm_pkg
      70              : // scope), UVM's reporting mechanism is accessible from anywhere
      71              : // outside ~uvm_component~, such as in modules and sequences.
      72              : // See <uvm_report_error>, <uvm_report_warning>, and other global
      73              : // methods.
      74              : //
      75              : //
      76              : // The ~uvm_top~ instance checks during the end_of_elaboration phase if any errors have
      77              : // been generated so far. If errors are found a UVM_FATAL error is being generated as result
      78              : // so that the simulation will not continue to the start_of_simulation_phase.
